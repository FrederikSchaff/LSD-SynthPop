Population Module

The population module can be loaded by putting the files into your work folder under the recommended structure Work/YOURMODEL/lsd-modules/pop/ and putting the line #include "lsd-modules/pop/backend_pop_lsd.h" in the header of your model file. Note: If you want to use the PajekFromCPP module to produce lineage networks, you need to include this module before including the current one.

On a global scope, a container holding the population-model class will be created. Thus, the default is to have one kind of population in a single model. This can, however, easily be generalised to allow for multiple populations (using the ADD_EXT capability of LSD, e.g.).

Currently the documentation only considers the macros. It is recommended to take a look at the example model to understand how this module works. As everything is under the MIT license, you may use and modify all of this in your own work, as long as you reference to the original module  This is how I understand the license and what my intention by choosing it is.

Currently there is only one statistical model available: BLL which refers to the simple parametric population model described in:

Boucekkine, Raouf; La Croix, David de; Licandro, Omar (2002): Vintage Human Capital, Demographic Trends, and Endogenous Growth. In Journal of Economic Theory 104 (2), pp. 340375. DOI: 10.1006/jeth.2001.2854.

You may also use the population module to only keep track of the heritage network and implement your own model in LSD, using the NONE model. If you use it this way, you can of course have different kinds of populations (and allow them to mix).

INIT_POPULATION_MODULE( "model", t_start, t_unit, par1, par2,  )

This Macro initialises the population module. Currently you may use two versions, BLL or NONE.

BLL: INIT_POPULATION_MODULE("BLL",0.0,1.0,alpha,beta)initializes the BLL model with parameters alpha, beta at time 0 and with time interval of size 1.0. In this model, you may use the macro POP_CONSTN_BIRTH(n) to get the necessary number of new agents at each time that supports an equilibrium population of size n. Each agent will get a death age according to the statistical model and dies when this age is reached. You still need to check this within LSD and delete the agent yourself, using the POP_ALIVE macro (Thus you may perform actions before deleting the LSD object).

NONE: Alternatively, INIT_POPULATION_MODULE("NONE",0.0,1.0) initializes only the heritage network model, i.e. there is neither a calculated reproduction rate nor a given time of death (it is set to -1 and hence cannot be reached). You need to manually kill a person, using POP_DIE_PERSON macro. Alternatively, you may provide a time of death when the person is created, using the POP_SET_DAGE macro.
